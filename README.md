# Лабораторная работа №12 - Проектирование классов. Базовая работа с ООП в Python

## Цели и задачи

### Цели
1. Освоить принципы объектно-ориентированного программирования (ООП) в Python.
2. Научиться проектировать и реализовывать классы, используя механизмы наследования, инкапсуляции и полиморфизма.
3. Развить навыки работы с объектами и взаимодействием между ними.
4. Создать базовый прототип системы управления транспортной логистикой с использованием классов.
5. Закрепить навыки практического применения ООП для решения задач реального мира.

### Задачи
1. Разработать базовый класс `Vehicle`, который будет использоваться в качестве родительского для всех видов транспортных средств.
2. Создать подклассы для различных транспортных средств (самолетов, фургонов) с уникальными атрибутами и методами.
3. Реализовать класс `TransportCompany`, включающий методы для добавления транспортных средств, клиентов и распределения грузов.
4. Написать методы для оптимизации загрузки транспортных средств, обеспечивающие минимальное использование ресурсов.
5. Проверить работу системы с тестовыми данными и оценить взаимодействие объектов.

## Классы

### 1. Client
Класс для представления клиентов транспортной компании.

**Атрибуты:**
- `name` - имя клиента
- `cargo_weight` - вес груза в тоннах
- `is_vip` - флаг VIP-статуса

**Методы:**
- `__str__()` - строковое представление клиента
- `__repr__()` - формальное строковое представление

### 2. Vehicle (базовый класс)
Базовый класс для всех транспортных средств.

**Атрибуты:**
- `vehicle_id` - уникальный идентификатор (генерируется автоматически)
- `capacity` - грузоподъемность в тоннах
- `current_load` - текущая загрузка
- `clients_list` - список клиентов, чьи грузы загружены

**Методы:**
- `load_cargo(client)` - загрузить груз клиента
- `unload_cargo()` - разгрузить транспорт
- `get_free_capacity()` - получить свободную грузоподъемность

### 3. Airplane
Класс самолета, наследуется от `Vehicle`.

**Дополнительные атрибуты:**
- `max_altitude` - максимальная высота полета в метрах

### 4. Van
Класс фургона, наследуется от `Vehicle`.

**Дополнительные атрибуты:**
- `is_refrigerated` - флаг наличия холодильника

### 5. TransportCompany
Класс для управления транспортной компанией.

**Атрибуты:**
- `name` - название компании
- `vehicles` - список транспортных средств
- `clients` - список клиентов

**Методы:**
- `add_vehicle(vehicle)` - добавить транспортное средство
- `add_client(client)` - добавить клиента
- `list_vehicles()` - получить список транспорта
- `optimize_cargo_distribution()` - оптимизировать распределение грузов
- `get_statistics()` - получить статистику компании

## Алгоритм оптимизации распределения грузов

Метод `optimize_cargo_distribution()` реализует следующий алгоритм:

1. **Сортировка клиентов:** VIP-клиенты обрабатываются первыми
2. **Сортировка транспорта:** транспорт сортируется по грузоподъемности (от большей к меньшей)
3. **Распределение:**
   - Для каждого клиента ищется транспорт с достаточным свободным местом
   - Сначала проверяется уже частично заполненный транспорт
   - Если не находится, проверяется пустой транспорт
4. **Приоритеты:** VIP-клиенты обслуживаются в первую очередь
